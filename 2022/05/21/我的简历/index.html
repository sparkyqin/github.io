项目描述

实现技术

Spring + Netty + Kubernetes + MyBatis + MySQL + Redis + RabbitMQ

项目职责

  • 作为该项目技术Leader，带领10+人开发团队，负责架构设计及核心代码开发，同时负责需求分解和项目推进
  • 采用容器化方案，解决环境搭建难且利用率低的问题。自研服务对接Kubernetes集群，实现定制化容器的管理接口，可在秒级启动所需镜像及配置的容器，提升环境获取效率50%以上；容器间共享宿主机资源，再加上定时清理任务，使得资源利用率提升80%；此外，挂载NFS实现容器空间扩展及共享
  • 采用增量化方案，解决代码合入时门禁执行全量UT导致耗时长的问题。设计思路为采集全量UT覆盖率信息，解析得到class与UT的映射关系，由此可在门禁时仅执行变更代码对应的UT来提升效率，采用Netty实现采集及解析服务来应对高并发上报场景，存储采用redis来提升数据写入和读取性能，同时，redis的set结构和过期时间天然地解决了数据去重和旧数据老化的问题，门禁耗时总体下降55%
  • 采用无侵入式插桩方案，解决线上问题定位难的问题。结合Java Agent和字节码增强技术，对运行时程序进行动态插桩，可在不修改源代码的情况下记录指定方法的运行时信息；使用TransmittableThreadLocal解决父子线程值传递问题，实现录制数据间的调用链解析；最终可使用FreeMarker和Jackson将录制数据生成为UT代码，实现问题定位与测试用例固化的双重目标

大流量实时通信平台（ 2018年12月 ~ 2020年11月）

项目描述

该项目为支撑华为畅连通话的信令交互云服务，拥有亿级用户容量，涉及中国、亚非拉、欧洲三大片区。

实现技术

Spring + Netty + MyBatis + MySQL + Redis + Kafka + Cassandra

项目职责

  • 作为项目核心开发人员，主要负责部署架构设计及核心代码开发，主导团队进行DevOps实践，不断改进工程能力，负责安排和指导印度团队工作，使其高效产出和自我成长
  • 主导服务端性能调优工作，提升吞吐量及资源利用率。对微服务进行多维度性能优化，具体措施包括JVM参数调优，充分利用内存并减少GC，MySQLc采用分库分表及读写分离架构，引入缓存并优化索引、连接数、减少写操作等，提升读写性能，使用JVisualVM观测压测状态下服务的cpu、线程状态及热点函数，寻找性能瓶颈，采用log4j2全异步模式提升日志输出性能，Kafka采用异步提交模式提升消息发送性能，最终整体性能提升3倍以上
  • 设计并实现漫游就近接入方案，解决隐私和时延问题。使用全局缓存解决信息同步问题，各代理服务定时上报自身信息至全局缓存，并定时更新对应归属地服务信息；使用netty解决互连问题，终端就近接入代理服务后，根据接入端口号从全局缓存获取所在地的真实服务信息并建立连接，代理服务关联两个连接并做消息透传，确保隐私安全，且代理服务和真实服务走专线网络可减少时延

其他项目

  • 负责电信网管补丁开发及运维工作，期间开发多款效率提升工具，包括SQL增量检查工具提升版本构建成功率，补丁合入工具提升补丁版本开发效率，Sqlserver工具提升单机版问题定位效率，案例地图提升现网问题解决效率
